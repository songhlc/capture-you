# Capture-Me 被动观察

## 触发条件
当用户发送任何消息时（不只是 `/capture-me` 命令），自动分析并提取画像信号。

## 信号提取规则

### 需要提取的维度（8个）
- **work**：工作相关信息（开会、项目、客户、汇报）
- **life**：日常生活（吃饭、睡觉、出行、购物）
- **habit**：习惯行为（每天、总是、经常、习惯）
- **emotion**：情绪状态（开心、焦虑、累、兴奋、压力）
- **preference**：偏好倾向（喜欢、讨厌、希望、想要）
- **goal**：目标计划（目标、打算、计划、决定）
- **relation**：人际关系（老婆、老公、家人、朋友、同事）
- **health**：健康状态（睡眠、运动、身体、疲惫）

### 提取标准
- 仅当消息包含明确的信号时提取（如"最近加班很累"包含 health + emotion）
- 短句或无意义内容不提取（如"好的"、"谢谢"）
- 每个维度置信度 0.5-1.0

### 调用方式
将信号写入 capture-me 数据库：

```bash
node ~/.claude/skills/capture-me/lib/observe-async.js '{"text":"最近加班很累","source":"claude-code"}'
```

### 频率控制
- 每条消息最多提取 2 个信号（避免刷屏）
- 同维度信号需间隔 5 分钟才重复记录
- 灵感/想法类信号不受此限制

### 不干扰原则
- 信号提取在后台异步执行
- 不打印任何输出到对话中
- 不影响正常对话响应
